07 — Rules

Hard rules for Claude Code

Not guidelines. These apply to every file, every component, every screen.

Do
Use #2A2760 as the primary brand color everywhere
Reserve #D93D2A for the single most important CTA per page
Use Fraunces only on marketing surfaces, never in app UI
Apply Inter at weights 400, 500, and 600 only
Use JetBrains Mono for all metrics, IDs, and handles
Sentence case everywhere — buttons, headings, nav, labels
Base all spacing on multiples of 4px using the token names
Use violet-tinted shadow values from the token specification
Keep border-radius ≤ 16px (pills use 9999px)
Write active voice, verb-first copy on all CTAs
Don't
Don't introduce any color outside the three defined ramps
Don't create gradients between brand and accent colors
Don't place two Primary or two Accent buttons in one viewport
Don't combine brand indigo and ember red in a single element
Don't use font-weight 700 or 800 anywhere in the product
Don't use pure #000000 or #FFFFFF as page backgrounds
Don't apply glassmorphism or backdrop-blur effects
Don't override token values with hardcoded hex in production
Don't use emoji as icons inside the app UI
Don't animate more than two CSS properties simultaneously
